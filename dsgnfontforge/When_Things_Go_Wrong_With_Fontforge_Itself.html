Книга о том, как создавать новые шрифты с помощью FontForge.

Посмотреть проект на GitHub

Когда что-то идет не так с самим FontForge

FontForge разработан на GitHub. Команда FontForge использует GitHub Issues для обсуждения проблем, ошибок и идей по улучшению, а затем кто-то разрабатывает решение и предлагает его как Pull Request.

Чтобы узнать больше о GitHub, посетить Хорошие ресурсы для изучения Git и GitHub.

Плата за поддержку

Это может быть неожиданностью, но платить за поддержку FontForge возможно и даже рекомендуется, если что-то пойдет не так.

В то время как другие редакторы шрифтов с аналогичными комплексными функциями стоят сотни долларов США, если каждый заплатит аналогичную сумму разработчикам FontForge за исправление самых неприятных ошибок, FontForge будет становиться все лучше и лучше.

СвободыеСпонсоры предлагается прозрачная система оплаты разработчикам FontForge за своевременное устранение волнующих проблем.

  1. Создать задачу FontForge, описывающую, что требуется изменить (смотреть ниже). Скопировать URL-адрес проблемы в буфер обмена.
  2. Посетить FreedomSponsors и спонсировать новый выпуск, используя URL-адрес, который скопирован ранее.
  3. Вернуться к проблеме и добавить комментарий со ссылкой на страницу проблемы FreedomSponsors с личным сообщением о том, что предлагается платная награда за закрытие этой проблемы.

Сообщить об ошибке

  1. Посетить FontForge GitHub Issue Tracker и войдите в GitHub (после создания учетной записи, если у вас ее еще нет).
  2. В поле поиска the Issues попробовать найти похожие проблемы, чтобы узнать, сообщалось ли уже о проблеме, с которой столкновение. Если это так, и проблема связана с ней, но не совсем совпадает, прокомментировать проблему, высказав свой собственный взгляд на нее.
  3. Если об этом еще не сообщалось, открыть новую проблему. Нажать зеленую кнопку “New Issue”, а затем описать вопрос, что сделано, чтобы вызвать сбой, или идею по улучшению.

Включить соответствующие детали, такие как:

  • операционная система и версия,
  • версия FontForge и откуда взята,
  • что происходит, шаг за шагом, чтобы создать проблему
  • какие сообщения об ошибках видны, и
  • какие ожидания, что происходит.

Перетащить скриншоты или другие изображения прямо на страницу задачи, чтобы включить их.

Самый простой способ сообщить о проблемах — записать видео-скринкаст, в котором закадровым голосом объяснть, что интересует, по мере их возникновения, а затем загрузить на YouTube и включить ссылку на видео.

Чтобы воспроизвести проблему, полезно поделиться с сообществом разработчиков файлами, с которыми работа. Если можно создать небольшой файл, содержащий только то, что необходимо для воспроизведения проблемы, создать репозиторий Fontforge и добавить эти файлы в /tests/fonts и отправить запрос на вытягивание. Также можно временно разместить файлы на веб-сайте или в файлообменной службе (например, MegaUpload, DropBox, Google Drive и так далее). Наконец, если нет желания публиковать файлы, указать адрес электронной почты, по которому разработчик FontForge сможет связаться и получить личную копию файла.

Пожалуйста, не закрывать проблемы других людей — попросить их закрыть проблему, если она закрыта к их удовлетворению.

Как сообщить о сбое

Процесс сообщения о сбое или других ошибках такой же, как и для новых функций или вопросов. Отправка хорошего отчета о сбое разработчикам FontForge действительно помогает им улучшить стабильность программы для всех! Не стесняться сообщать о таких проблемах, потому что сбой, о котором не сообщается, — это сбой, который вряд ли удастся исправить.

Если обнаружить сбой FontForge во время использования, создать проблему, как указано выше. Если есть определенный файл шрифта (SFD, UFO, OTF, TTF и так далее), что вызывает сбой, можно либо загрузить его в новый репозиторий GitHub самостоятельно (или в Dropbox, либо на другую платформу) и добавить ссылку, либо опубликовать свое электронное письмо. и попросить разработчика отправить электронное письмо, чтобы получить копию конфиденциально.

По описанию разработчики FontForge попытаются воспроизвести сбой. Если они смогут это сделать, то смогут определить, где в коде идет ошибка, и исправить ситуацию.

После объединения запроса на включение, устраняющего проблему, потребуется получить новую версию. Сделать одно из следующих действий:

  • перекомпилировать из последнего исходного кода GitHub (смотреть Инсталяция Fontforge),
  • проверить, доступна ли ежедневная сборка (часто это возможно для Mac OS X), или
  • дождаться следующего выпуска (часто в течение нескольких недель).

Лучшие отчеты о сбоях

Чтобы помочь разработчикам выяснить, что происходит не так, и really понять, как это исправить, можно проделать еще немного работы и создать файл backtrace. Обратная трассировка включает в себя список того, какие программные функции вызывали какие другие, чтобы добраться до места, где программа перестала работать. Обратная трассировка наиболее полезна, если она также содержит номера строк функций.

Чтобы выполнить обратную трассировку, может потребоваться установка из исходного кода с включенным debugging information. Использовать команды type и nm, чтобы найти путь и статус двоичного файла Fontforge. Пример:

$ type -all fontforge;
fontforge is /usr/bin/fontforge
$ nm /usr/bin/fontforge;
nm: /usr/bin/fontforge: no symbols

В этом примере видно no symbols, поэтому необходимо обновить установку, включением в нее отладочную информацию.

Установить отладочную информацию в Fedora

Fedora предлагает в стандартном репозитории команду для простой установки отладочной информации для FontForge. (Обратить внимание, что для загрузки могут потребоваться сотни мегабайт, если еще не установлено множество зависимых пакетов debuginfo.) Чтобы установить его, запустить:

debuginfo-install fontforge;

Обратная трассировка создается с помощью отладчика проекта GNU, gdb. Можно либо подключить gdb к уже работающему FontForge, либо запустить FontForge внутри самого сеанса gdb. Вот пример последнего:

Одно из основных отличий, которое дает запуск FontForge внутри gdb, заключается в том, как становится очевидным сбой. Без GDB при сбое FontForge он исчезнет с экрана. Однако когда запускать FontForge внутри gdb, сбой FontForge останется открытым вместе со своими окнами и пользовательским интерфейсом.

Если обнаружить, что интерфейс не отвечает, вернуться к терминалу, на котором запущен gdb, и можно увидеть в тексте что-то вроде SIGSEGV, за которым следует приглашение (gdb). Если видно приглашение (gdb), значит, FontForge больше не выполняется.

В этом примере разработчик может увидеть, что FontForge произошел сбой внутри функции copy(). Функция copy() сама была вызвана из функции KCD_AutoKernAClass. Обратная трассировка сообщит разработчику программного обеспечения, в каких именно строках были сделаны эти вызовы, а также воспользуется подсказкой о том, что параметр, переданный в copy(), был недействителен (выходит за пределы), чтобы выяснить, что код делает неправильно.
